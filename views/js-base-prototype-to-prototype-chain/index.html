# prototype

什么是原型 prototype? 引用《Javascript高级程序设计》6.2.3原型模式一句话就是：

每个函数都有一个 prototype(原型) 属性，这个属性是一个指针，指向一个对象，而这个对象的用途是包含可以由指定类型的所有实例共享的属性和方法。

我们从例子来说明

// 1-2
function Foo() {
  // ...
}
Foo.prototype.name = 'revan'
var foo1 = new Foo()
var foo2 = new Foo()
console.log(foo1.name) // revan
console.log(foo2.name) // revan

1-2函数的 prototype 属性指向了一个对象，这个对象正是调用该构造函数创建实例的原型。

那么原型可以理解来说：每一个 Javascript 对象(不包括null)在创建的时候会与之关联另一个对象，这个关联的对象就是原型，每个对象都会从原型“继承”属性。

# 实例与原型

当读取实例的属性时，如果找不到，就会查找与之关联的原型中的属性，如果还查不到，就查找原型的原型，一直到最顶层为止。

举个例子：

function Foo() {
  
}
Foo.prototype.name = 'Amber'
var foo = new Foo()

foo.name = 'revan1i'
console.log(foo.name)
delete foo.name
console.log(foo.name)

# 原型链

那么 Object.prototype 的再上一级原型是什么？

是 null 通过 console.log(Object.prototype.__proto__ === null) // true 就可以验证得知。

null 引用 阮一峰老师的《http://www.ruanyifeng.com/blog/2014/03/undefined-vs-null.html》说法是：

null 表示“没有对象”， 即此处不应该有值。

所以，Object.prototype.__proto__ 值为 null 表示 Object.prototype 没有原型。查找属性的时候查到 Object.prototype 就可以停止查找了。

# __proto__

__proto__ 是浏览器提供的非标准的属性用于访问原型，绝大部分浏览器都支持这个属性。然而它并不存在于 Foo.prototype 中，实际上，它来自于 Object.prototype , 与其说是一个属性，不如说是一个 getter/setter ，当使用 obj.__proto__ ，可以理解成返回了 Object.getPrototypeOf(obj)。

# 真的是继承吗？

最后是关于继承，前面我们讲到“每一个对象都会从原型‘继承’属性”，实际上，继承是一个十分具有迷惑性的说法，引用《你不知道的JavaScript》中的话，就是：

继承意味着复制操作，然而 JavaScript 默认并不会复制对象的属性，相反，JavaScript 只是在两个对象之间创建一个关联，这样，一个对象就可以通过委托访问另一个对象的属性和函数，所以与其叫继承，委托的说法反而更准确些。
